# LessonProof design direction

LessonProof is a review workspace, not a marketing template. The interface must make the correction, cited evidence, proposed patch, human approval, and deterministic release proof easy to inspect in that order.

## Visual direction

- Warm editorial canvas, dark ink, one clay accent, and semantic colors only for release state.
- The native system sans stack for interface text and Georgia italics for one restrained display phrase. The production page makes no third-party font request.
- A 1240px workbench with a compact header, short introduction, release gate, safety strip, and a twelve-column workflow.
- The active stage receives the most space. Completed stages collapse but remain reviewable. Future stages stay visibly locked to the sequence.
- Borders establish grouping. Surfaces stay flat.

## Anti-slop rules

- No gradients.
- No `box-shadow`.
- No emoji or text glyphs used as interface icons.
- No decorative blobs, glass panels, neon glows, or generic three-card marketing grids.
- No pill shape by default. Reserve compact rounded labels for status or provenance.
- No vague copy such as “unlock,” “empower,” “seamless,” “leverage,” “streamline,” “robust,” “cutting-edge,” “elevate,” “harness,” or “delve.”
- Buttons name the action and consequence.
- Success green is reserved for checks that passed and releases that were verified. Human approval is not presented as verification.

## Reference boundary

The direction re-creates abstract editorial and review-workspace patterns from a separate project. It does not import that project’s code, logo, photographs, client material, copy, or proprietary font files.
